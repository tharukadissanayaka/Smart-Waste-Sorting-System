# Model Evaluation

This directory contains the evaluation artifacts generated by running the model on the test dataset.

## Generating the Outputs
To regenerate these outputs, run the evaluation script from the repository root:
```bash
python scripts/evaluate.py
```

## Contents
When the evaluation script runs, it uses the Ultralytics built-in validation suite. It will generate and save several output files here, including:
- **`confusion_matrix.png` / `confusion_matrix_normalized.png`**: Highlights misclassifications between visually similar classes (e.g., clear plastic vs. glass).
- **`PR_curve.png`**: Precision-Recall curves for each class, showing the tradeoff between precision and recall at different confidence thresholds.
- **`F1_curve.png`**: F1 confidence curve.
- **`P_curve.png`** and **`R_curve.png`**: Precision and Recall curves across confidence thresholds.

**Note on Checkpoints:**
If the actual trained model (`models/waste_yolov8_best.pt`) is not present or not fully trained yet, the evaluation script may fall back to the pretrained `yolov8n.pt` base model. In this case, the outputs and metrics generated here will not reflect true model performance. Once the fine-tuned checkpoint is available, simply re-run the evaluation script.
